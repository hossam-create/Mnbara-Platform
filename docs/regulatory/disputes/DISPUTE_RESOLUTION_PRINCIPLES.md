## 1. Constitutional Declaration

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   DISPUTES ARE RESOLVED BY HUMANS. NEVER BY AI.            │
│                                                             │
│   Artificial Intelligence may assist with:                  │
│   • Organizing evidence                                     │
│   • Summarizing timelines                                   │
│   • Suggesting relevant policies                            │
│                                                             │
│   AI SHALL NEVER:                                           │
│   • Determine dispute outcomes                              │

│   • Auto-resolve disputes                                   │
│   • Override human judgment                                 │
│                                                             │

│                                                             │

```

## 2. Core Principles

### 2.1 Human Authority

| Principle | Implementation |
| :--- | :--- |
| **Human decides** | Every outcome requires human judgment |
| **Human accountable** | Named reviewer for every dispute |
| **Human explains** | Decisions include human-written rationale |
| **Human reviewable** | Appeals reviewed by different human |

### 2.2 Fairness

| Principle | Implementation |
| :--- | :--- |
| **Both parties heard** | Opportunity to present evidence |
| **No prejudgment** | Decision after review, not before |
| **Consistent standards** | Same rules for all parties |
| **Proportional outcomes** | Response matches severity |

### 2.3 Transparency

| Principle | Implementation |
| :--- | :--- |
| **Process disclosed** | Users know how disputes work |
| **Evidence visible** | Parties see what was considered |
| **Decision explained** | Rationale provided in writing |
| **Timeline clear** | Expected timeframes communicated |

### 2.4 Finality with Appeal

| Principle | Implementation |
| :--- | :--- |
| **Decisions are binding** | After appeal exhausted |
| **Appeal right exists** | Every party can appeal once |
| **Clear endpoint** | Final decision is final |
| **Enforcement follows** | Outcomes are implemented |

## 3. AI Role Boundaries

### 3.1 What AI May Do (Advisory Only)

| Function | Scope | Human Oversight |
| :--- | :--- | :--- |
| Organize evidence | Sort and tag submissions | Human reviews organization |
| Summarize timeline | Create chronological view | Human verifies accuracy |
| Flag inconsistencies | Highlight contradictions | Human evaluates significance |
| Suggest policies | Identify relevant rules | Human decides applicability |
| Priority scoring | Estimate urgency | Human confirms priority |
| Similar case lookup | Find precedents | Human evaluates relevance |

### 3.2 What AI SHALL NEVER Do

| Prohibited Function | Reason |
| :--- | :--- |
| **Determine outcomes** | Human judgment required |
| **Auto-resolve disputes** | Violates human authority |
| **Deny evidence** | Human decides admissibility |
| **Close disputes** | Human closes |
| **Communicate decisions** | Human communicates |
| **Calculate compensation** | Human determines amounts |
| **Override reviewer** | Human has final word |

### 3.3 Enforcement

Any AI system that attempts to:
- Make binding dispute decisions
- Auto-resolve disputes
- Override human reviewers

Will be:
1. Immediately disabled
2. Investigated as Critical Incident
3. Reported to Board

## 4. Dispute Categories

### 4.1 Category Definitions

| Category | Description | Example |
| :--- | :--- | :--- |
| **Item Not Received** | Buyer paid, item never arrived | Package never delivered |
| **Not As Described** | Item differs materially from listing | Wrong model, wrong color |
| **Damaged in Transit** | Item arrived damaged | Broken screen, dented case |
| **Partial Delivery** | Some items missing | 2 of 3 items received |
| **Late Delivery** | Delivered after deadline | Missed holiday deadline |
| **Payment Issue** | Payment not received/processed | Escrow release failure |
| **Cancellation Dispute** | Disagreement on cancellation | Who caused cancellation |
| **Fraud Allegation** | Suspected fraudulent activity | Fake item, stolen goods |

### 4.2 Severity Classification

| Severity | Criteria | Resolution Timeline |
| :--- | :--- | :--- |
| **Standard** | <$500, clear facts | 5 business days |
| **Elevated** | $500-$2,000 OR complex facts | 10 business days |
| **High** | $2,000-$10,000 OR safety concern | 15 business days |
| **Critical** | >$10,000 OR fraud OR legal | 20 business days |

## 5. Resolution Standards

### 5.1 Evidence Standard

| Standard | Application |
| :--- | :--- |
| **Preponderance of evidence** | More likely than not (>50%) |
| **Both parties' evidence** | Must consider all submissions |
| **Objective evidence preferred** | Photos, tracking, timestamps |
| **Credibility assessment** | Human evaluates believability |

### 5.2 Outcome Options

| Outcome | Description |
| :--- | :--- |
| **Full refund** | Buyer receives 100% back |
| **Partial refund** | Buyer receives partial amount |
| **No refund** | Buyer claim denied |
| **Split resolution** | Both parties share loss |
| **Item return required** | Refund upon return |
| **Traveler penalty** | Traveler fee reduced/forfeited |
| **Account action** | Warning, limitation, or restriction |

### 5.3 Remedies

| Party | Available Remedies |
| :--- | :--- |
| **Buyer** | Refund, partial refund, replacement |
| **Traveler** | Payment release, fee reinstatement |
| **Both** | Mutual cancellation |

## 6. Timeline Commitments

### 6.1 Process Timeline

| Stage | Standard | Elevated/High | Critical |
| :--- | :--- | :--- | :--- |
| Acknowledge receipt | 24 hours | 24 hours | 4 hours |
| Initial review | 48 hours | 72 hours | 48 hours |
| Request evidence | 72 hours | 96 hours | 72 hours |
| Evidence deadline | 5 days | 7 days | 10 days |
| Final decision | 5 days | 10-15 days | 20 days |

### 6.2 Extension Rules

| Scenario | Extension | Approval |
| :--- | :--- | :--- |
| Party requests more time | Up to 5 days | Reviewer |
| Complex investigation | Up to 10 days | Manager |
| External dependency | Case-by-case | Senior Manager |

## 7. Quality Standards

### 7.1 Decision Requirements

Every dispute decision **MUST** include:

| Element | Description |
| :--- | :--- |
| **Decision statement** | Clear outcome |
| **Evidence considered** | List of materials reviewed |
| **Reasoning** | Why decision was reached |
| **Policy basis** | Which rules applied |
| **Appeal rights** | How to appeal |
| **Implementation** | What happens next |
| **Reviewer identity** | Name/ID of decider |

### 7.2 Quality Metrics

| Metric | Target |
| :--- | :--- |
| Decision within SLA | >95% |
| Appeal rate | <15% (healthy) |
| Overturn rate | <10% |
| User satisfaction (post-dispute) | >70% |
| Consistency score | >90% |

## 8. Prohibited Practices

### 8.1 Reviewer Prohibitions

Dispute reviewers **SHALL NOT**:

| Prohibited | Reason |
| :--- | :--- |
| Decide without evidence review | Due process |
| Favor repeat users | Equal treatment |
| Consider trust score as determinative | Not a factor in facts |
| Communicate before decision | Process integrity |
| Accept external pressure | Independence |
| Decide disputes involving self | Conflict of interest |

### 8.2 Process Prohibitions

| Prohibited | Reason |
| :--- | :--- |
| Auto-close without decision | User expectation |
| Default to one party | Fairness |
| Maximum favor to platform | User trust |
| Hidden criteria | Transparency |

## 9. Governance

### 9.1 Oversight Structure

| Level | Responsibility |
| :--- | :--- |
| Dispute Reviewer | Initial decision |
| Senior Reviewer | Elevated cases, appeals |
| Dispute Manager | Quality, escalations |
| Trust Review Board | Policy appeals, precedents |
| Chief Risk Officer | Governance oversight |

### 9.2 Policy Updates

Changes to dispute principles require:
1. Risk Committee proposal
2. Legal review
3. Board approval
4. 30-day user notice
