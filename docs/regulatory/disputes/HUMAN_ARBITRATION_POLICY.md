## 1. Policy Statement

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   EVERY DISPUTE IS DECIDED BY A QUALIFIED HUMAN.           │
│                                                             │
│   "Qualified" means trained, certified, and accountable.    │
│                                                             │
│   AI NEVER DECIDES. AI ONLY ASSISTS.                        │
│                                                             │
│   The human reviewer:                                       │
│   • Reads all evidence personally                           │
│   • Applies judgment to facts                               │
│   • Writes the decision rationale                           │
│   • Takes accountability for outcome                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. Arbitrator Qualifications

### 2.1 Dispute Reviewer (Level 1)

**Authority:** Standard disputes up to $500

| Requirement | Standard |
| :--- | :--- |
| Training | 40-hour dispute resolution course |
| Certification | Pass platform certification exam |
| Supervision | First 30 decisions reviewed |
| Background | Clean background check |
| Recertification | Annual |

### 2.2 Senior Reviewer (Level 2)

**Authority:** Elevated disputes up to $2,000, First-level appeals

| Requirement | Standard |
| :--- | :--- |
| Experience | 6 months as Level 1, 100+ decisions |
| Training | 80-hour advanced training |
| Certification | Senior certification exam |
| Quality record | <8% overturn rate |
| Recertification | Annual |

### 2.3 Dispute Manager (Level 3)

**Authority:** High-value disputes up to $10,000, Second-level appeals

| Requirement | Standard |
| :--- | :--- |
| Experience | 1 year as Level 2, 300+ decisions |
| Training | Management training + mediation certification |
| Quality record | <5% overturn rate |
| Leadership | Supervisory experience |
| Recertification | Annual |

### 2.4 Trust Review Board Member (Level 4)

**Authority:** Critical disputes, Policy appeals, Precedent setting

| Requirement | Standard |
| :--- | :--- |
| Experience | 2+ years dispute management |
| Appointment | Nominated by CRO, approved by CEO |
| Independence | No direct operational role |
| Term | 2-year term, renewable |

## 3. Decision-Making Process

### 3.1 Mandatory Human Steps

Every dispute decision **REQUIRES** these human actions:

```
MANDATORY HUMAN DECISION PROCESS

Step 1: EVIDENCE REVIEW
□ Reviewer personally reads all submitted evidence
□ Reviewer views all photos/documents
□ Reviewer reviews transaction history
□ AI may organize; human must read

Step 2: FACT FINDING
□ Reviewer identifies disputed facts
□ Reviewer weighs conflicting claims
□ Reviewer determines what happened
□ AI may highlight; human must decide

Step 3: POLICY APPLICATION
□ Reviewer identifies applicable policies
□ Reviewer determines how policies apply to facts
□ Reviewer reaches preliminary conclusion
□ AI may suggest; human must apply

Step 4: DECISION
□ Reviewer makes final determination
□ Reviewer documents reasoning
□ Reviewer confirms outcome is proportional
□ Human decides alone

Step 5: COMMUNICATION
□ Reviewer drafts decision communication
□ Decision includes all required elements
□ Appeal rights clearly stated
□ Human writes; AI may assist with formatting only
```

### 3.2 AI Assistance Boundaries

| Stage | AI May | AI Shall Not |
| :--- | :--- | :--- |
| Evidence | Organize, tag, summarize | Exclude, weight, prioritize |
| Fact Finding | Highlight discrepancies | Determine facts |
| Policy | Suggest relevant rules | Apply rules to facts |
| Decision | Provide templates | Make or influence decision |
| Communication | Format text | Write substantive content |

## 4. Independence Requirements

### 4.1 Reviewer Independence

| Requirement | Implementation |
| :--- | :--- |
| No party relationship | Cannot decide if knows party |
| No prior involvement | Cannot decide if previously involved |
| No financial interest | No stake in outcome |
| No pressure | Decision without interference |
| Conflict disclosure | Report any potential conflict |

### 4.2 Conflict of Interest Rules

| Scenario | Action |
| :--- | :--- |
| Knows either party | Mandatory recusal |
| Previous transaction with party | Mandatory recusal |
| Colleague involved | Avoid if possible |
| Prior decision on same user | Allowed (not a conflict) |
| Dispute involves reviewer's market | Manager review |

### 4.3 Anti-Influence Rules

Reviewers **SHALL NOT**:

| Prohibited | Consequence |
| :--- | :--- |
| Accept direction on outcome | Disciplinary action |
| Consider platform financial impact | Removal |
| Favor high-value users | Disciplinary action |
| Discuss pending disputes externally | Termination |

## 5. Accountability Framework

### 5.1 Individual Accountability

| Element | Implementation |
| :--- | :--- |
| Named reviewer | Every decision attributed to person |
| Decision record | Full history of reviewer's decisions |
| Quality metrics | Individual quality tracked |
| Outcome tracking | Appeals and overturns tracked |

### 5.2 Quality Review

| Review Type | Frequency | Reviewer |
| :--- | :--- | :--- |
| Random sample | 10% of decisions weekly | Manager |
| Appealed decisions | 100% | Senior reviewer |
| Overturned decisions | 100% + coaching | Manager |
| New reviewer | 100% for first 30 | Senior reviewer |

### 5.3 Performance Standards

| Metric | Target | Action if Below |
| :--- | :--- | :--- |
| Timeline compliance | >95% | Coaching |
| Quality score | >85% | Retraining |
| Overturn rate | <10% | Review |
| User satisfaction | >70% | Analysis |
| Policy compliance | 100% | Immediate review |

## 6. Workload Management

### 6.1 Capacity Standards

| Standard | Limit | Rationale |
| :--- | :--- | :--- |
| Maximum decisions/day | 15 standard, 8 elevated | Quality assurance |
| Minimum review time | 15 min standard, 30 min elevated | Thoroughness |
| Break requirements | 15 min per 2 hours | Fatigue prevention |

### 6.2 Queue Management

| :--- | :--- | :--- |

| P2: Urgent | Time-sensitive, elevated | Same day |

| P4: Low | Minor, no urgency | FIFO |

## 8. Escalation Procedures

### 8.1 When to Escalate

| Trigger | Escalate To |
| :--- | :--- |
| Exceeds authority level | Next level reviewer |
| Novel situation | Manager for guidance |
| Potential fraud | Fraud team |
| Legal concern | Legal team |
| Media/reputation risk | Communications + Manager |
| Safety concern | Trust & Safety |

### 8.2 Escalation Process

```
ESCALATION PROCEDURE

1. Document reason for escalation
2. Summarize case facts
3. Note any time sensitivity
4. Transfer to appropriate party
5. Notify original parties of transfer
6. Remain available for questions
```
