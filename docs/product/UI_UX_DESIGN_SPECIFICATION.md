## 4.1 Tone Definition

### Voice Attributes
| Attribute | Description | Example |
| :--- | :--- | :--- |
| **Calm** | No urgency, no pressure, no FOMO | "Take your time" not "Act now!" |
| **Trust-First** | Lead with safety, transparency | "Your money is protected" |
| **No Pressure** | User sets the pace | "There's no rush to decide" |
| **No Legal Language** | Plain, human words | "We'll help" not "Subject to terms" |

### What We Sound Like
```
âœ… "Your money goes to escrow until you confirm delivery."
âœ… "This traveler is new. They're still building history."
âœ… "Take your time to review the offers."
âœ… "You decide what's fair."
```


## 4.3 Microcopy Patterns

### Field Labels
```
Instead of: "Item Name"
Use: "What's the item called?"

Instead of: "Enter store location"  
Use: "Where can they buy it?"

Instead of: "Traveler compensation"
Use: "What will you pay the traveler?"
```

### Error Messages
```
Instead of: "Invalid input"
Use: "Please fill this in to continue."

Instead of: "Payment failed"
Use: "Payment didn't go through. Check your card or try another method."

Instead of: "User not found"
Use: "We couldn't find that account. Check the spelling?"
```

### Confirmation Messages
```
Instead of: "Action completed successfully"
Use: "Done! Your [action] is complete."

Instead of: "Transaction processed"
Use: "Your money is safely in escrow."
```

# APPENDIX: DESIGN CHECKLIST

Before shipping any screen, verify:

**Layout & Hierarchy**
- [ ] Most important action is clearly primary
- [ ] Generous whitespace (not crowded)
- [ ] Clear visual grouping
- [ ] Mobile-first responsive design

**Trust Elements**
- [ ] TrustBadge visible on all user cards
- [ ] Trust reasons accessible (tap to see)
- [ ] RiskBanner shown when applicable
- [ ] Escrow explanation present in payment flows

**AI Transparency**
- [ ] All suggestions have "Why" explanations
- [ ] "You decide" messaging present
- [ ] No auto-execution of any kind
- [ ] Override always possible

**Copy & Tone**
- [ ] No urgency language
- [ ] No legal jargon
- [ ] Plain, human words
- [ ] Calm, trust-first voice

**Anti-Patterns Check**
- [ ] No countdown timers
- [ ] No scarcity messaging ("only X left")
- [ ] No hidden fees
- [ ] No gamification elements
- [ ] No complex animations
